# ERP Agent ID Hypotheses

Черновая гипотеза по сопоставлению `agent_id` из ERP `POST /tickets/by-sell`
с каналами продаж из weekly-таблиц.

Статус:
- это не финальный mapping;
- использовать как рабочую гипотезу для сверок;
- часть соответствий подтверждена сильнее, часть пока только по паттерну сумм и косвенным признакам.

Период наблюдения:
- март 2026;
- дополнительно использовались недельные срезы `2026-03-09`, `2026-03-16`, `2026-03-23`, `2026-03-30`.

## Working Notes

- `agent_id = 0` не нужно смешивать с денежными каналами при сверке выручки.
- Для уверенного финального mapping полезно отдельно собрать:
  - частотность `client_name` по `agent_id`;
  - частотность `agent_id` по неделям;
  - сопоставление с weekly-строками по суммам и count;
  - при наличии в ERP справочник агентов или endpoint со списком агентских каналов.

## Current Practical Mapping

Черновая рабочая версия для быстрых сверок:

```text
39992173 -> афиша.ру
39995745 -> МДТЗК
39320770 -> билеты виджет (сайт)
39320755 -> билеты яндекс афиша
39996012 -> кассир / внешний агрегатор
0        -> бесплатные / служебные / нулевые билеты
```

## Caveat

Последние три строки пока нельзя считать полностью подтвержденными.
Если будем строить автоматический channel mapping в БД, лучше хранить еще и `confidence`
или отдельный флаг `is_hypothesis`.
